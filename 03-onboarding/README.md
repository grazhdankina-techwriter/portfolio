# 🚀 Онбординг: Первые шаги Технического писателя

> 📋 **Паспорт проекта**
> * **Цель:** Ускорить адаптацию нового сотрудника и стандартизировать процессы.
> * **ЦА:** Новые технические писатели / методисты в IT-компании.
> * **Формат:** Онбординг-гайд, SOP (Стандартная операционная процедура), Глоссарий.

Добро пожаловать в команду! Этот гайд поможет тебе быстро освоиться, получить доступы и понять, как мы работаем с документацией.

## 📅 Чек-лист на первую неделю

### День 1: Знакомство и доступы
- [ ] Пройти встречу с HR и тимлидом.
- [ ] Получить корпоративную почту и доступ к Slack/Telegram.
- [ ] Запросить доступы: Jira, Confluence, GitHub, Figma.
- [ ] Изучить Брендбук и Гайдлайны по написанию текстов (ссылка).

### День 2-3: Погружение в продукт
- [ ] Пройти демо-версию продукта как обычный пользователь.
- [ ] Изучить текущую пользовательскую документацию (Help Center).
- [ ] Познакомиться с Product Manager'ом и узнать о ближайших релизах.

### День 4-5: Первые задачи
- [ ] Завести свой первый тикет в Jira (например, "Обновить скриншоты в разделе Настройки").
- [ ] Написать черновик статьи для внутренней базы знаний.
- [ ] Пройти ретроспективу первой недели с ментором.

## 📖 SOP: Как запросить доступ к репозиторию на GitHub

**Шаг 1.** Откройте Jira и найдите тикет вашего проекта.

**Шаг 2.** В комментариях к тикуту оставьте запрос по шаблону:
> *Привет! Прошу предоставить доступ (Read/Write) к репозиторию [название_репо]. Мой GitHub-ник: @ваш_ник.*

**Шаг 3.** Поставьте тег `@TeamLead` или `@DevOps`.

**Шаг 4.** После получения приглашения на почту, примите его и проверьте, что можете клонировать репозиторий локально.

## 📚 Глоссарий (Словарь терминов)

| Термин | Определение |
| :--- | :--- |
| **Деплой (Deploy)** | Процесс переноса кода или документации с локальной среды на продакшн (продакшн-сервер). |
| **Прод (Production)** | Боевая среда, с которой работают реальные пользователи. |
| **Мерж-реквест (Merge Request / PR)** | Запрос на слияние изменений в коде/документации из одной ветки в другую. Требует ревью. |
| **Ревью (Review)** | Проверка текста или кода коллегой перед его финальным утверждением. |
| **SOP** | Standard Operating Procedure — стандартная операционная процедура, пошаговый регламент. |
